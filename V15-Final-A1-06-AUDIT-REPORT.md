# V15-Final-A1-06 감사 보고서

## 범위
CSS·JavaScript·렌더링 우선순위·이미지 로딩 속성·동작 접근성을 전체 167개 HTML에서 점검했습니다. 기존 구조는 잠금 상태로 유지했습니다.

## 실제 반영
- 홈 첫 화면의 CSS 배경 이미지(`/assets/hero-technician.jpg`)를 브라우저가 CSS 분석 전에 발견하도록 preload를 추가했습니다.
- 이미지 567개를 검사해 누락된 `decoding="async"` 513개를 HTML에 정적으로 반영했습니다. JavaScript 실행 이후가 아니라 HTML 파싱 단계부터 적용됩니다.
- 첫 화면에 위치한 지역 랜딩·시공사례 대표 이미지 54개는 `loading="eager"`와 `fetchpriority="high"`로 조정했습니다. 아래쪽 이미지는 기존 lazy loading을 유지했습니다.
- 모션 감소를 설정한 사용자에게 불필요한 전환·애니메이션을 최소화하는 CSS를 추가했습니다.
- CSS와 JavaScript 캐시 버전을 `V15-Final-A1-06`으로 갱신했습니다.

## 점검 결과
- 외부 JavaScript: 전 페이지 `defer` 유지
- 인라인 실행 스크립트: 구조화 데이터(JSON-LD) 외 별도 블로킹 코드 없음
- 공통 CSS: 약 45.3KB
- 공통 JavaScript: 약 2.0KB
- 별도 폰트 다운로드: 없음
- 구조 변경: 없음
- 검증 오류: 0개

## 의도적으로 보류
- 이미지 원본의 크기·포맷·width/height 최적화는 실제 원본이 필요한 작업이므로 계획한 이미지 단계에서 처리합니다.
- 사용하지 않는 것으로 보이는 `styles.css`는 현재 HTML에서 참조되지 않지만, 기존 배포 호환성을 위해 삭제하지 않았습니다.
- GitHub Pages는 저장소의 `_headers` 파일을 일반적인 응답 헤더 설정으로 적용하지 않으므로, 이번 단계에서는 캐시 파일명 버전 갱신으로 안전하게 대응했습니다.

## Structural Lock
메뉴, URL, 폴더, 페이지 수, 레이아웃, 본문, 이미지 경로, 동영상은 변경하지 않았습니다.
